# Funūn Contract Formatting Defaults

Use these defaults unless the user chooses a different direction. These defaults are for Funūn specifically and should not be treated as a generic music-contract brand system.

## Brand posture

Contracts should feel premium and intentional, not loud.

- Use `Funūn`, not `Funun`
- Favor white-page legal-document presentation over dark app mimicry
- Use accent color sparingly
- Keep the hierarchy clear and calm

## Visual direction

- White page
- Black or near-black body text
- Subtle Funūn accent in headings or a thin divider
- Indigo / fuchsia accent only in a restrained way
- Soft gray-lavender labels for helper copy if the destination format supports it cleanly

## Structural defaults

For most music contracts:

1. Title
2. Subtitle or purpose line
3. Work / project details
4. Party or contributor table
5. Agreement language
6. Signature blocks
7. Short helper note where needed

## Field helper defaults

Use these when the user needs fallback guidance:

- `Artist Name`
  - `Enter the intended primary release artist name. If not final, use the working artist name for this project.`
- `Record Label`
  - `Enter the release label. If self-releasing, enter "Independent". If undecided, use "TBD" only temporarily.`
- `Legal Name`
  - `Use your full legal name exactly as registered with your PRO.`
- `PRO / Society`
  - `Enter ASCAP, BMI, SESAC, SOCAN, or the applicable society.`

## Split-sheet-specific defaults

- Recommended title: `Funūn Split Sheet`
- Recommended subtitle: `Songwriter and Publishing Split Confirmation`
- Prefer a clean split table over manual numbered blank lines
- Give each signer a distinct block for:
  - legal name
  - split percentage
  - PRO / society
  - publishing designee
  - administrator
  - signature
  - date
