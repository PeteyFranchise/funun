# Contract Intake Rubric

Use this rubric when reviewing a blank contract template for Funūn.

## 1. Identify the document type

Classify the contract first:

- split sheet
- work-for-hire
- producer agreement
- collaboration agreement
- license
- release / consent
- other music-business form

State the document type plainly before giving design advice.

## 2. Separate substance from presentation

Treat these as likely legal substance:

- ownership allocation
- payment or royalty language
- amendment / modification clauses
- signature / assent language
- governing definitions
- rights granted or retained

Treat these as likely presentation:

- title formatting
- section order when substance stays intact
- labels and helper text
- table structure
- whitespace, rules, headings, and form layout

Treat these as user guidance:

- what to enter when details are unknown
- legal-name consistency reminders
- PRO / IPI helper copy
- self-release / label fallback instructions

## 3. Review for common problems

Flag:

- outdated or generic titles
- ambiguous blanks
- duplicated fields
- weak signer instructions
- poor grouping of metadata
- manual underscore lines that should be tables or structured fields
- places where users will not know what to enter

## 4. Recommend, do not overreach

When the request is formatting-first:

- improve structure
- improve readability
- add safe helper text
- leave legal meaning intact

Only recommend substantive clause rewrites when the user explicitly asks.
