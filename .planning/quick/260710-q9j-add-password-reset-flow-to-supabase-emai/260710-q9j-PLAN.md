# Quick Task 260710-q9j: Password-Reset Flow + Auth Docs

## Objective

Close the one real gap in the otherwise-complete Supabase email/password auth: there is
no password-reset flow and no "Forgot password?" affordance. Add the reset request page,
the new-password page, wire the recovery redirect through the existing `/auth/callback`,
harden the callback for failures, extend middleware coverage, and document auth setup +
testing. Preserve all existing working auth (signup, signin, signout, email-confirmation
callback, route protection) — do NOT rebuild them.

Resend SMTP (`noreply@auth.funun.studio` via `smtp.resend.com:465`) is already configured
in the Supabase dashboard; the app only needs to call the right Supabase auth methods.

## Tasks

### Task 1 — Password reset request page + signin link
**Files:** `app/(auth)/forgot-password/page.tsx` (new), `app/(auth)/signin/page.tsx` (edit)
**Action:**
- New `'use client'` `/forgot-password` page matching `signin/page.tsx` styling (same `inputClass`,
  `rounded-xl` card, error/success blocks). Email input → `supabase.auth.resetPasswordForEmail(email,
  { redirectTo: \`${baseUrl}/auth/callback?next=/update-password\` })`, `baseUrl =
  process.env.NEXT_PUBLIC_APP_URL ?? window.location.origin`. Non-enumerating success state
  ("If an account exists for that email, we've sent a reset link."). "Back to sign in" link.
- Add a "Forgot password?" `Link` to `/forgot-password` on the signin page near the password
  field, styled like the existing secondary links. No other signin logic change.
**Verify:** `npx tsc --noEmit` clean; grep confirms `resetPasswordForEmail` and the `/forgot-password` link.

### Task 2 — New-password (recovery) page
**Files:** `app/(auth)/update-password/page.tsx` (new)
**Action:** `'use client'` `/update-password` page, same styling. New + confirm password fields
(minLength 6, must match). Verifies a recovery session exists (getSession + PASSWORD_RECOVERY
`onAuthStateChange` fallback for hash-fragment links); calls `supabase.auth.updateUser({ password })`.
Success state → link to `/signin` + auto-redirect to `/vault`. No-session/expired-link state →
message pointing back to `/forgot-password`.
**Verify:** `npx tsc --noEmit` clean; grep confirms `updateUser` and `onAuthStateChange`.

### Task 3 — Callback hardening + middleware coverage
**Files:** `app/auth/callback/route.ts` (edit), `middleware.ts` (edit)
**Action:**
- Callback: capture `exchangeCodeForSession` error; on failure redirect to
  `/forgot-password?error=recovery` when `next` is `/update-password`, else `/signin?error=auth`.
  Preserve the successful `?code=` → `next` path (drives both email confirmation and recovery).
- Middleware: treat `/forgot-password` as a public auth route (bounce to `/vault` if fully
  signed in); keep `/update-password` reachable during the temporary recovery session (NOT an
  auth-route bounce target, NOT protected).
**Verify:** `npx tsc --noEmit` clean; grep confirms error handling + both new paths in middleware.

### Task 4 — Auth documentation
**Files:** `README.md` (edit/create `## Authentication` section)
**Action:** Document required env vars (`NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`,
`SUPABASE_SERVICE_ROLE_KEY`, `NEXT_PUBLIC_APP_URL`), Supabase redirect-allowlist requirement,
signup test steps, password-reset test steps, and Resend/Supabase email-delivery verification.
Note sender `noreply@auth.funun.studio` (Funún) via `smtp.resend.com:465`.
**Verify:** README contains the `## Authentication` heading and all four env var names.

## must_haves

**truths:**
- A signed-out user can open `/forgot-password`, submit their email, and Supabase sends a recovery email via the configured Resend SMTP.
- The recovery email link lands the user on `/update-password` with an active recovery session, where they set a new password via `updateUser`.
- The signin page exposes a "Forgot password?" link to `/forgot-password`.
- `/auth/callback` redirects to a safe error page on a failed/expired code exchange instead of silently landing on `/vault` with no session.
- Existing signup, signin, signout, email-confirmation, and route protection are unchanged and still compile.

**artifacts:**
- `app/(auth)/forgot-password/page.tsx`
- `app/(auth)/update-password/page.tsx`
- (edited) `app/(auth)/signin/page.tsx`, `app/auth/callback/route.ts`, `middleware.ts`, `README.md`

**key_links:**
- forgot-password page → `supabase.auth.resetPasswordForEmail(..., { redirectTo: .../auth/callback?next=/update-password })`
- `/auth/callback` → exchanges recovery code → redirects to `/update-password`
- update-password page → `supabase.auth.updateUser({ password })`
