# Quick Task 260711-2nt: Fix Next.js Manifest Build Regression

## Objective

Every Vercel deployment on this project (all PRs, all branches, going back weeks) fails with:

```
Error: ENOENT: no such file or directory, lstat
'/vercel/path0/.next/server/app/(admin)/page_client-reference-manifest.js'
```

This is [vercel/next.js#93862](https://github.com/vercel/next.js/issues/93862) ŌĆö a Next.js
15.5.x regression where a route with zero client-component boundaries in its tree never gets
its `page_client-reference-manifest.js` written, but Next's own build-trace step still expects
the file to exist and crashes. Confirmed: does NOT reproduce in a local `next build` (same
Next.js version, 15.5.19) ŌĆö Vercel-environment-specific, not caused by any application code in
this milestone. Confirmed via `Learn about Vercel status page`: no active Vercel infrastructure
incident; this is code/route-shape specific.

`app/(admin)/page.tsx` (bare `redirect('/admin/checklist')`) and `app/(admin)/layout.tsx` have
no `'use client'` anywhere in their tree (confirmed via grep) ŌĆö exactly the shape that triggers
the bug.

**Fix approach:** cheap, reversible workaround (not a Next.js version change). Give the
`(admin)` route group one trivial client-component boundary so Next.js is forced to emit the
manifest.

## Task

**Files:** `components/admin/ManifestBoundary.tsx` (new), `app/(admin)/layout.tsx` (edit)

**Action:**
1. Create `components/admin/ManifestBoundary.tsx`:
   ```tsx
   'use client'
   export function ManifestBoundary() {
     return null
   }
   ```
2. Read the existing `app/(admin)/layout.tsx` first. Import `ManifestBoundary` and render it
   alongside `{children}` (not replacing anything) ŌĆö this covers every current/future route in
   the `(admin)` group (`/`, `/admin/checklist`, `/admin/members`, `/admin/curators`, `/tips`),
   not just the one redirect page that surfaced it.

**Verify:**
- `npx tsc --noEmit` ŌĆö must be clean.
- `npm run build` ŌĆö must exit 0. Note: this bug has not reproduced locally, so a clean local
  build does not fully prove the fix; the real test is a fresh Vercel Preview deployment on a
  PR, which happens after this task (out of scope for this quick task itself).

## Out of scope
- No `package.json`/`package-lock.json` changes (no Next.js version change this task).
- No changes to phase-15 files or the `auth-password-reset-flow` branch ŌĆö this fix ships on its
  own branch/PR off `origin/main`, same pattern as the auth-reset PR (#27).

## must_haves

**truths:**
- `app/(admin)/layout.tsx` renders `<ManifestBoundary />`, giving the `(admin)` route group a
  real client-component boundary.
- `npx tsc --noEmit` and `npm run build` both pass locally with no behavior change to any
  existing route.

**artifacts:**
- `components/admin/ManifestBoundary.tsx`
- (edited) `app/(admin)/layout.tsx`
