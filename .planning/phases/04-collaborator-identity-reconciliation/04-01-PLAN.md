## Phase Goal

**As a** music collaborator who was credited on someone else's project before I had a Funūn account, **I want to** have my past contributions automatically linked to my new account when I sign up, **so that** I can see everything I'm credited on without anyone re-entering my data.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /api/claim-collaborators | Untrusted POST; must derive identity from the session cookie, never from the body or headers |
| middleware → /api/claim-collaborators | Server-to-server fire; forwards the session cookie so the route re-validates |
| SECURITY DEFINER function → collaborators (other users' rows) | Intentional cross-user write; only legitimate inside the DB function, never from app code with a user-session client |
| client → /collaborators credits query | Cross-user read; must be row-scoped by RLS to `claimed_by = auth.uid()` |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-01 (T-claim-spoof) | Elevation of Privilege | POST /api/claim-collaborators | high | mitigate | Route validates `session.user.id` via `createApiClient().auth.getUser()`; user id/email taken only from the session, never from request body or custom header |
| T-04-02 (T-mass-assign) | Tampering | claim_collaborators / handle_new_user | high | mitigate | `claimed_by` is written only inside SECURITY DEFINER functions; it is excluded from `COLLABORATOR_EDITABLE_FIELDS`, so no user-session PATCH can set it |
| T-04-03 (T-rls-leak) | Information Disclosure | "Claimed users see own credits" RLS policy | high | mitigate | Policy is FOR SELECT USING `auth.uid() = claimed_by` — row-scoped to the claiming user only, never table-wide |
| T-04-04 (T-backfill) | Tampering | claim_collaborators back-fill block | medium | mitigate | Every back-fill UPDATE uses `COALESCE(existing_column, new_value)` with the existing value first — additive-only, never overwrites artist-entered data |
| T-04-05 | Denial of Service | middleware claim hook | low | mitigate | Hook is fire-and-forget with a `.catch()`; gated to fire only while `claimed_at IS NULL`, then short-circuits to avoid per-request DB work on the hot path |
| T-04-SC | Tampering | npm/pip/cargo installs | high | mitigate | No new packages installed this phase (RESEARCH Package Legitimacy Audit); no install task present |
</threat_model>
