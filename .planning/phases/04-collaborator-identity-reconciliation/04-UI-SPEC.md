## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding (`px-1.5 py-0.5` pattern) |
| sm | 8px | Compact element spacing, tight row gaps (`gap-2`) |
| md | 16px | Default element spacing, card internal padding (`p-4`) |
| lg | 24px | Section padding, card grid gap (`gap-6`) |
| xl | 32px | Layout gaps between major blocks (`mt-8`) |
| 2xl | 48px | Major section breaks (`py-12` empty states) |
| 3xl | 64px | Page-level top/bottom padding (`py-10`) |

Exceptions:
- Touch targets on ghost buttons: minimum 32px height (`min-h-[32px]`) — matches Phase 1 CollaboratorCard Edit button pattern
- Favorite star touch target: minimum 32×32px to meet interactive element minimum

Source: `CollaboratorCard.tsx`, `CollaboratorRoster.tsx`, `dashboard/page.tsx` spacing patterns

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label / caption | 12px–12.5px | 700 (bold) | 1.4 |
| Heading (page / section) | 20px–22px | 800 (extrabold) | 1.2 |
| Display (stat values) | 24px (`text-2xl`) | 600 (semibold) | 1.2 |

Notes:
- Secondary/muted text uses `text-lavdim` (`#7c80b4`) at 13–14px / weight 400
- Badge labels use 10px (`text-[10px]`) at weight 700 — reserved for status chips only (IPI, Funūn member)
- Section sub-headings follow `text-lg font-semibold text-white` (18px/600) — matches dashboard `Recent projects` pattern
- Tab labels use `text-sm font-medium` (14px/500) — matches `ProjectTabs.tsx`

Source: `CollaboratorCard.tsx` (14.5px name, 12.5px PRO), `CollaboratorRoster.tsx` (22px h1 extrabold), `dashboard/page.tsx` (24px stat, 18px section head), `ProjectTabs.tsx` (14px tabs)

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0a0a0f` (`ink`) | Page background — all page surfaces |
| Secondary (30%) | `#0E0D1E` (`card`) + `#1A1838` (`card2`) | Card backgrounds, picker dropdown panel, tab rail, raised chips |
| Accent (10%) | `#818CF8` (`brandindigo`) → `#D946EF` (`brandfuchsia`) gradient | CTA buttons, Funūn-member badge, "Add new collaborator" text link in picker, favorite star active state |
| Destructive | `#F43F5E` (`rose`) | Delete confirmation text and hover states only — no other use |

Accent reserved for:
1. Primary CTA buttons (gradient `bg-grad` — `Add collaborator`, `Add your first collaborator`)
2. `Funūn member` badge background and text (`bg-brandindigo/10 text-brandindigo border-brandindigo/30`)
3. `IPI on file` badge (`bg-brandindigo/10 text-brandindigo`)
4. Picker bottom action "Add new collaborator" text (`text-brandindigo`)
5. Favorite star when active (filled, `text-brandindigo`)
6. Tab active underline — white (`#ffffff`), not accent (matches existing `ProjectTabs` pattern)

Semantic colors (non-accent):
- `#F59E0B` (`money` / amber): IPI-missing warning badge (`bg-amber-400/10 text-amber-300 border-amber-400/30`) — reuse existing Phase 1 pattern for Archive warning affordance
- `#7c80b4` (`lavdim`): muted/secondary text, inactive nav, empty state body
- `#C7CBF7` (`lav`): secondary text (not used as background)
- Hairline borders: `rgba(199,203,247,.12)` (`hair`) for card borders; `rgba(199,203,247,.22)` (`hairstrong`) for dropdown panels

## Interaction Contracts

### Favorite Star Toggle (CollaboratorCard)

- Single click: PATCH `/api/collaborators/[id]` with `{ is_favorite: !current }`. Optimistic UI — flip icon immediately, revert on error.
- Loading state: star icon `animate-pulse opacity-50`.
- No confirmation required.

### Archive (Claimed Collaborator)

- Click Archive button on card: show inline confirmation within the card — replace card content with: `"Remove [Name] from your active roster? Their credits are preserved." [Archive] [Cancel]` — same inline pattern as the existing split-sheet row delete pattern.
- On confirm: PATCH `/api/collaborators/[id]/archive`. Card moves to archived group. `router.refresh()` after success.
- No full-page modal needed — inline confirm within card bounds.

### Unarchive

- In Archived view: each archived card shows an `Unarchive` ghost button at bottom-right. No confirmation. PATCH `/api/collaborators/[id]/unarchive`. Card moves back to active roster.

### Credits Tab Navigation

- URL query param persistence: `/collaborators?tab=credits` and `/collaborators?tab=roster`. Allows deep-linking from dashboard "View all" link.
- Tab switching is client-side (`useState`) but initial tab set from URL search param on mount.

### CollaboratorPicker — Favorite Toggle

- Within the picker dropdown, each roster row shows a star icon on hover (not visible at rest to avoid clutter).
- Click star: PATCH favorite in background, update local picker state. No dropdown close.

## States Required

Each new or modified surface must handle these states:

### CollaboratorCard

| State | Trigger |
|-------|---------|
| Default | `claimed_by IS NULL`, `archived_at IS NULL` |
| Claimed | `claimed_by IS NOT NULL` |
| Favorite | `is_favorite === true` |
| Archived | `archived_at IS NOT NULL` (only shown in Archived toggle view) |
| Archive confirming | User clicked Archive |

### Credits Section (My Credits tab)

| State | Trigger |
|-------|---------|
| Loading | Server fetch in progress (SSR — handled at page level, no skeleton needed) |
| Empty | `credits.length === 0` |
| Populated | `credits.length > 0`, first 20 rows |
| Paginating | User clicked "Show more" — client fetch, append rows |

### Dashboard Credits Preview

| State | Trigger |
|-------|---------|
| Hidden | `claimed credits === 0` — section not rendered |
| Visible | `claimed credits > 0` — max 3 rows shown |

### Picker Dropdown

| State | Trigger |
|-------|---------|
| No groups | `roster.length === 0` — shows "Add collaborator" trigger (existing) |
| Grouped (Favorites + Recent + All) | `roster.length > 0`, no search active |
| Flat results | Search query entered — groups collapsed |
| No search results | `filtered.length === 0` — shows "No results" (existing) |

## Accessibility Contract

- All interactive elements with icon-only interaction (star, archive) must have `aria-label` set: `aria-label="Add to favorites"`, `aria-label="Archive collaborator"`.
- Tab switcher (`My Credits` / `My Roster`): `role="tablist"` on container, `role="tab"` on each button, `aria-selected`, `role="tabpanel"` on each panel — matches existing `ProjectTabs.tsx` pattern.
- Archived cards: `aria-hidden={true}` on control buttons while in read-only archived display.
- Credits table rows: use `<ul>` / `<li>` for list semantics. Each row's split sheet link has descriptive `aria-label="View split sheet for [Song Name]"`.
- Inline archive confirmation: focus moves to the "Archive" confirm button on open. Escape key or "Keep" restores normal card view.
