Complete the shared-document approval loop and the collaborator invite flow on top of the standalone split sheets from Plan 03. <threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| anonymous browser → /approve/[token] + POST /api/approve/[token] | Unauthenticated, public token endpoints — the token is the only secret |
| anonymous browser → /join/[inviteToken] | Public view-only profile keyed by an invite token |
| initiator → other users' party rows | send-for-approval writes rows visible to other accounts (service-client, RLS bypass) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-10 | Spoofing | /approve/[token], /join/[inviteToken] | high | mitigate | 256-bit `crypto.randomBytes(32)` tokens make brute-force infeasible; token in URL is the CSRF defense for the account-less form |
| T-01-11 | Tampering | POST /api/approve/[token] counter-proposal | high | mitigate | Server re-validates counter_split ∈ [0,100]; status transitions guarded (already-final tokens rejected) per Pitfall 2 |
| T-01-12 | Elevation of Privilege | send-for-approval service-client writes | high | mitigate | Ownership verified via `.eq('initiator_user_id', user.id)` BEFORE any createServiceClient call; service client never used on unverified input |
| T-01-13 | Elevation of Privilege | stale token reuse (approve + invite) | medium | mitigate | `token_expires_at` checked on every public render and POST; used/final tokens render the expired state |
| T-01-14 | Information Disclosure | /approve and /join public pages | medium | mitigate | Public pages expose only name + split % (approve) or the recorded profile to the invited party (join); no other-user PII beyond name/% |
| T-01-15 | Denial of Service | repeated invite emails | low | mitigate | 24h cooldown per collaborator+artist pair suppresses duplicate sends (Pitfall 4) |
| T-01-SC | Tampering | npm installs | low | accept | No new packages introduced this phase (RESEARCH.md: zero new dependencies); tokens use Node built-in crypto |
</threat_model>
