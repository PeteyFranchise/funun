## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /api/collaborators | Untrusted JSON body crosses into the collaborator CRUD layer |
| user A → user B data | Cross-user read/write attempts on the global roster |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-01 | Information Disclosure | GET /api/collaborators, collaborators table | high | mitigate | RLS policy `USING (auth.uid() = user_id)` plus `.eq('user_id', user.id)` filter in the GET handler |
| T-01-02 | Tampering | POST/PATCH /api/collaborators | high | mitigate | sanitizeCollaborator() with COLLABORATOR_EDITABLE_FIELDS allowlist strips unknown keys (mass-assignment defense); id/user_id/timestamps never sourced from body |
| T-01-03 | Elevation of Privilege | PATCH/DELETE /api/collaborators/[id] | high | mitigate | Handler chains `.eq('id', params.id).eq('user_id', user.id)`; RLS WITH CHECK is the backstop |
| T-01-04 | Spoofing | All collaborator routes | medium | mitigate | `createApiClient()` + `supabase.auth.getUser()` 401 gate on every handler |
| T-01-SC | Tampering | npm installs | low | accept | No new packages introduced this phase (RESEARCH.md: zero new dependencies); no install task to gate |
</threat_model>
