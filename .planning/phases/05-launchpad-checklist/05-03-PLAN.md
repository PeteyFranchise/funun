<objective>
Build the two artist-facing API routes that power the per-project Launchpad room: a GET that returns checklist items (ordered by sort_order, with tip gating) merged with the calling user's progress, and a PATCH that toggles a single item's completion. These satisfy LAUNCH-02 (week-ordered items + action CTAs flow through), LAUNCH-03 (only approved tips reach artists), and LAUNCH-04 (completion persists per user/project).

Purpose: The artist UI (Plan 04) consumes these endpoints. The tip-gating logic lives here so unapproved drafts never leave the server.
Output: GET checklist route + PATCH progress route.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-01 | Elevation of Privilege | progress PATCH | high | mitigate | user_id is forced to the session `user.id`; never read from the body. RLS `WITH CHECK (auth.uid() = user_id)` rejects any mismatched write at the DB layer |

| T-05-SC | Tampering | npm installs | low | accept | No package installs in this plan |
