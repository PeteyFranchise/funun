## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-01 | Elevation of Privilege | launchpad_progress rows | high | mitigate | RLS policy `USING (auth.uid() = user_id) WITH CHECK (auth.uid() = user_id)` restricts every row to its owner; enforced in this migration |
| T-05-03 | Tampering | orphaned progress rows after item delete | low | mitigate | FK `item_key REFERENCES launchpad_checklist_items(key) ON DELETE CASCADE` removes progress rows atomically when an item is deleted |
| T-05-SC | Tampering | npm installs | high | mitigate | No package installs in this plan; dnd-kit install + legitimacy audit handled in Plan 02 (all three packages already audited OK in RESEARCH.md § Package Legitimacy Audit) |
</threat_model>
