| Property | Value |

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline chip padding |
| sm | 8px | Compact element spacing, row internals |
| md | 16px | Default element spacing, panel body padding |
| lg | 24px | Section padding, card internal spacing |
| xl | 32px | Layout gaps between sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:

- Checklist item rows: `px-[18px] py-4` (matches existing playbook row on `/launchpad`) — 18px horizontal is a project-established exception for card rows
- Topbar: `px-9 pb-[22px] pt-[26px]` — project standard, apply unchanged to `/launchpad/[projectId]`
- TipPanel width: `max-w-md` (448px) — matches `ToolSidePanel` exactly; do not change
- Checkbox touch target: 44px minimum — the checkbox element must be at least 44×44px tappable; pad the hit area with `p-2.5` if the visual is smaller
- Admin drag handle: 44px row height minimum for dnd-kit drag targets on touch

Source: spacing values derived from existing `launchpad/page.tsx` and `ToolSidePanel.tsx` patterns.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0a0a0f` (`ink`) | Page background, TipPanel backdrop |
| Secondary (30%) | `#0E0D1E` (`card`) / `#1A1838` (`card2`) | Checklist item cards, TipPanel surface, admin list rows |
| Accent (10%) | `#818CF8` → `#D946EF` (`grad`) | Specific elements listed below |

Accent reserved for:
1. Active nav indicator bar (left edge of nav item) — already established pattern
2. Gradient progress badge on per-project room header (e.g. "3 of 12 complete" completion fraction text uses `.gtext` class)
3. Section "Algorithm window" callout banner border/tint — `border-brandindigo/30 bg-[linear-gradient(150deg,rgba(129,140,248,.14),rgba(217,70,239,.08))]` (matches existing info banner pattern from `LaunchpadPage`)
4. Admin "Add item" primary CTA button — `bg-grad` gradient fill

Secondary semantic colors (from project palette, not accent):
- Completion checkmark fill: `emerald-400` (`#34D399`) — checked item state
- Unchecked checkbox border: `white/20` — inactive state
- "Before release" collapsed confirmation block tint: `amber-400/10 border-amber-400/20` — retrospective context
- Admin approve button: `emerald-400` text on `emerald-400/10` background
- Admin reject button: `rose-400` text on `rose-400/10` background
- No-tip-available notice in TipPanel: `lavdim` text, no special color

Source: color values from `tailwind.config.ts`, `globals.css`, and existing component patterns.

## Component Interaction Contracts

### ChecklistItem row

- Full row is clickable (excluding checkbox) — opens TipPanel
- Clicking checkbox area toggles completion only — does NOT open TipPanel
- Checkbox target area: minimum 44×44px (`p-2.5` wrapper around the visible 20px checkbox)
- Completed item: label gets `line-through text-white/40`; checkbox shows emerald-400 fill with white checkmark SVG
- Hover state (non-completed): `hover:bg-white/5` on the row
- Active/selected (panel open for this item): `bg-card2 border-hairstrong`

### ChecklistSection collapse (Before release)

- Section header has a chevron toggle button (right side)
- Collapsed state: shows header + sub-label + a "Did you handle this?" confirmation block listing items as compact checkboxes (no row click → panel interaction in collapsed mode)
- Expanded state: shows full item rows as normal
- Section auto-collapses when project `release_date` is in the past and completion < 100%
- Collapse animation: `transition-all duration-200` on the content area height

### TipPanel

- Slides in from the right: `translate-x-full` → `translate-x-0` with `transition-transform duration-200`
- Full-height fixed panel: `fixed inset-y-0 right-0 z-50 w-full max-w-md`
- Backdrop: `fixed inset-0 bg-black/60 backdrop-blur-sm` — click closes
- Escape key closes
- Header: item label (bold, 18px) + section badge + close button (matches ToolSidePanel header pattern exactly)
- Body sections in order: tip body text (if tip_approved), step-by-step list, action CTA button
- No-tip state: show only step-by-step + CTA; no empty tip placeholder visible
- CTA button: internal tool link → `bg-white text-black font-semibold`; external URL → same style with external arrow icon appended
- Footer pinned at bottom: `border-t border-white/10 p-5`

### Admin ChecklistAdmin (dnd-kit drag rows)

- Drag handle icon (6-dot grip) on left of each row, `cursor-grab` / `cursor-grabbing`
- Dragging row: `opacity-60 shadow-2xl scale-[1.02]`
- Drop target: `border-2 border-brandindigo/60` highlights the target slot
- Mobile fallback: up/down arrow buttons replace drag; arrows are `rounded-lg border border-white/15 p-1.5 text-white/50 hover:text-white` buttons
- Delete: shows destructive confirmation inline below the row ("Delete this item? This removes it and all artist progress on it permanently. [Cancel] [Delete item]") — no modal, inline expand
- Add/Edit form: inline expand below the list (not a modal); same inline-expand pattern as delete confirm

### Admin TipsAdmin

- Each draft row is a card (`bg-card border-hair rounded-[14px] p-4`)
- Tip text is editable in a `textarea` — auto-expands; `resize-none` but grows with content via JS
- Approve button: `emerald-400/10 border-emerald-400/30 text-emerald-400` pill
- Reject button: `rose-400/10 border-rose-400/30 text-rose-400` pill
- Approved state: row gets `bg-emerald-400/[0.04] border-emerald-400/20`; buttons replaced with "Approved" badge
- Author field (`ai` / `admin`) and drafted date shown as `text-[11px] text-lavdim` metadata below the tip text

### ProjectCards (on global `/launchpad` page)

- Cards added ABOVE existing playbook content, separated by a `border-b border-hair mb-7` divider
- Card grid: `grid grid-cols-2 gap-3` on ≥640px, single column below
- Each card: `rounded-[14px] border border-hair bg-card px-[18px] py-4 flex items-center justify-between` — matches existing playbook task row width/radius
- Card content: project title (`font-bold text-white`) + completion fraction badge (`text-[12px] text-lavdim`)
- CTA: "Open Launchpad →" link (`text-[13.5px] font-bold text-white bg-card2 rounded-[10px] px-4 py-[10px]`)

## States and Visibility Rules

| State | Behavior |
|-------|----------|
| Checklist item: incomplete | Checkbox unchecked (`border border-white/20 rounded`), label full opacity |
| Checklist item: complete | Checkbox filled emerald (`bg-emerald-400`), white checkmark, label `line-through text-white/40` |
| Checklist item: tip panel open | Row background `bg-card2 border-hairstrong` |
| Before release section: release date past | Auto-collapse to confirmation view; chevron shows collapsed state |
| Before release section: no release date | Show expanded, no auto-collapse |
| TipPanel: approved tip exists | Show: tip body → steps → CTA |
| TipPanel: no approved tip | Show: steps → CTA (skip tip body block entirely) |
| TipPanel: no steps and no tip | Show: CTA only (fallback to item label as instruction) |
| Admin tip row: pending | Normal card, editable textarea, Approve + Reject buttons visible |
| Admin tip row: approved | `bg-emerald-400/[0.04]` tint, "Approved" badge, buttons hidden |
| Admin tip row: rejected | `bg-rose-400/[0.04]` tint, "Rejected" badge, buttons hidden; not shown in the main list (filter to pending only by default; rejected archived below a "Show rejected" toggle) |
| Admin checklist row: dragging | `opacity-60 scale-[1.02] shadow-2xl` |
| Admin checklist row: drag target slot | `border-2 border-brandindigo/60` between rows |

## Accessibility Contracts

- Checkbox: `role="checkbox"` `aria-checked={completed}` on the button element
- TipPanel: focus trap when open; first focusable element is the close button; Escape closes
- Section collapse toggle: `aria-expanded={isOpen}` `aria-controls="{sectionId}-items"`
- Drag handles: `aria-label="Drag to reorder {item label}"` — dnd-kit accessible via `@dnd-kit/accessibility`
- Mobile up/down arrow buttons: `aria-label="Move {item label} up"` / `"Move {item label} down"`
- Delete confirmation: `role="alert"` on the inline expand so screen readers announce it
- Admin approve/reject buttons: `aria-label="Approve tip for {item key}"` / `"Reject tip for {item key}"`
