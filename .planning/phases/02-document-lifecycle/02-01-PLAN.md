    Create app/api/vault/[projectId]/documents/[docId]/upload/route.ts.

    Export async function POST(request: Request, { params }: { params: Promise<{ projectId: string; docId: string }> }).

    Steps:
    1. Destructure projectId and docId from await params.
    2. Create supabase client with createApiClient(). Get user with supabase.auth.getUser(). If no user return NextResponse.json({ error: 'Unauthorized' }, { status: 401 }).
    3. Parse multipart: const formData = await request.formData(). Extract file = formData.get('file') as File | null. If !file || !(file instanceof File) return NextResponse.json({ error: 'No file provided' }, { status: 400 }).
    4. Verify the document belongs to this user and project — before uploading, confirm ownership: const { data: existing } = await supabase.from('vault_documents').select('id, status').eq('id', docId).eq('project_id', projectId).eq('user_id', user.id).maybeSingle(). If !existing return NextResponse.json({ error: 'Document not found' }, { status: 404 }).
    5. Call uploadSignedPdf({ file, userId: user.id, projectId, docId }). Wrap in try/catch — on error return NextResponse.json({ error: err.message }, { status: 400 }) for validation errors (PDF check, size check) or { status: 500 } for storage errors.
    6. Update the vault_documents row atomically: supabase.from('vault_documents').update({ status: 'signed', signed_at: new Date().toISOString(), file_url: url, signed_by: user.email ?? user.id }).eq('id', docId).eq('project_id', projectId).eq('user_id', user.id).select().single(). On error return { status: 500 }.
    7. Return NextResponse.json({ data: doc }).

    Import from: 'next/server' (NextResponse), '@/lib/supabase/server' (createApiClient), '@/lib/vault/documents' (uploadSignedPdf).

    The route intentionally does NOT accept a status field in the body — uploading a PDF IS the signing action. This prevents the manual "mark signed" pattern for upload-backed documents.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /api/vault/[projectId]/documents/[docId]/upload | Untrusted multipart body with a user-supplied file crosses into document storage |
| user A → user B documents | Cross-user read/write attempts on vault_documents |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-01 | Spoofing | POST /upload route | high | mitigate | createApiClient() + supabase.auth.getUser() 401 gate before any file access |
| T-02-02 | Tampering | vault_documents update | high | mitigate | Triple .eq('id', docId).eq('project_id', projectId).eq('user_id', user.id) ownership check before upload and again on the update; RLS USING (auth.uid() = user_id) is the backstop |
| T-02-03 | Information Disclosure | release-documents storage bucket | medium | mitigate | Files stored under userId/ path prefix; getPublicUrl is acceptable for signed PDFs (user chose to upload them); RLS on vault_documents row gates who can see file_url |
| T-02-04 | Denial of Service | PDF file size | medium | mitigate | MAX_DOC_SIZE = 5MB enforced in uploadSignedPdf() before storage call; Next.js default body limit applies to formData parsing |
| T-02-05 | Tampering | File type spoofing | medium | mitigate | MIME type check on file.type === 'application/pdf' in uploadSignedPdf(); Content-Type validated before storage write |
| T-02-SC | Tampering | npm installs | low | accept | No new packages introduced this plan; zero new dependencies |
