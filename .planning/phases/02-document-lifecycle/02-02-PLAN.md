## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-06 | Tampering | DocumentCard handleFileChange | medium | mitigate | Client validates accept="application/pdf" (UX only); server-side MIME check in uploadSignedPdf() is the enforced gate |
| T-02-07 | Information Disclosure | file_url public link | low | accept | Signed PDFs are intentionally uploaded by the artist; the URL is rendered only to authenticated users who own the document; URL is a long random Supabase storage path with no predictable structure |
| T-02-08 | Spoofing | documentId in fetch URL | medium | mitigate | Upload route verifies ownership via triple .eq before accepting the file; a spoofed docId returns 404 |
| T-02-SC | Tampering | npm installs | low | accept | No new packages introduced; no install task to gate |
</threat_model>
