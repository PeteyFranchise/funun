# Phase 14 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope Note (read first)

Per `14-CONTEXT.md` D-14: **no dedicated visual redesign pass is needed.** The existing 3-column `PlaybackView.tsx` layout already broadly matches `docs/design/wave-4-social-layer/playback.html`'s target design. This contract governs only the **new** surfaces this phase adds, built directly into the existing structure:

1. The readiness-score widget, two placements (topbar chip + inline near tracklist/files) — D-02
2. Stems (ZIP) upload/download affordance + info (ⓘ) button — D-03/D-04/D-07/D-09
3. Instrumental upload affordance — D-03/D-05
4. The reworked Master/Instrumental playback toggle — D-06
5. Consistent empty-state hiding across all of the above — D-08
6. The Export Pack button + delivery panel (direct download vs. shareable link) — D-10/D-11/D-12

D-01 (project card → playback room link target) is a routing change with no new visual surface — not covered here.

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — hand-rolled Tailwind (project-wide convention, confirmed across all 8 prior shipped phases; no `components.json`, no shadcn anywhere in this codebase) |
| Icon library | inline SVG (hand-drawn paths) — matches `PlaybackView.tsx`'s existing transport-control icon convention and `playback.html`'s checkmark/box-icon glyphs |
| Font | `var(--font-sans)`, system-ui, sans-serif (`tailwind.config.ts`) |

**Why no shadcn (not re-asked this phase):** This decision has been made identically in all 8 prior shipped phases (see `08-UI-SPEC.md`, `05/06/07-UI-SPEC.md`) — hand-rolled Tailwind against the project's custom design tokens (`ink`/`card`/`card2`/`lav`/`lavdim`/`brandindigo`/`brandfuchsia`/`hair`/`money`) is the established, locked visual system. Introducing shadcn for one phase would fork the system. Continuing the established pattern.

## Spacing Scale

`PlaybackView.tsx` already ships a locked spacing system (D-14: no redesign). Per the 8-point-scale ideal, the closest-fit token per existing value:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps in Files rows; waveform bar gaps (`gap-[3px]`, existing exception) |
| sm | 8px | Button icon gaps (`gap-2`), chip internal padding, Files-row vertical rhythm (`mt-2`) |
| md | 16px | New Files-row entries (Instrumental, Stems) follow the existing `mt-2`/`text-[13px]` row rhythm |
| lg | 24px | Section-to-section spacing within a column (`mt-5`/`mt-6` on toggle/waveform) |
| xl | 32px | Not newly introduced this phase |
| 2xl | 48px | Not newly introduced this phase |
| 3xl | 64px | Not newly introduced this phase |

**Exceptions (existing `PlaybackView.tsx` values — preserved verbatim per D-14, not redesigned):**
- `px-9 py-[30px] pb-[110px]` — page container padding (accounts for the fixed mini-player)
- `gap-7` (28px) — 3-column grid gap
- `p-5` (20px) — card padding (tracklist, credits, metadata panels)
- `py-[10px]` — track row vertical padding

New elements added by this phase (readiness widget, upload rows, Export Pack panel) must reuse one of the above existing values — do not introduce a new spacing value.

## Typography

`PlaybackView.tsx` already ships a locked type scale (D-14: no redesign). Declared sizes for **new** elements this phase adds:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Section label (e.g. "Files", "Export pack" panel headers) | 12px (`text-[12px]`, uppercase, `tracking-[.16em]`) | 700 bold | 1.4 |
| Body / row label (Files rows, panel copy, dialog text) | 13–13.5px (`text-[13px]`/`text-[13.5px]`) | 400 regular (labels) / 700 bold (values, buttons) | 1.5 |
| Chip / button label (readiness chip, Export pack button, Download stems button) | 12.5–13px (`text-[12.5px]`/`text-[13px]`) | 700 bold | 1.3 |
| Panel/dialog title ("Export pack") | 18px (`text-[18px]`, existing weight step below the 26px hero title) | 700 bold | 1.2 |

Weights used: **400 regular** (row labels, meta/secondary text, info-button body copy) and **700 bold** (section headers, buttons, chips, panel titles, values). This matches `PlaybackView.tsx`'s existing bold/regular split for UI chrome — do not introduce `font-semibold`/`font-medium` for new elements (the existing 600/800 weights on track titles and the hero "now playing" title are pre-existing and untouched by this phase).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0a0a0f` (`ink`) | Page background, mini-player bar background |
| Secondary (30%) | `#0E0D1E` (`card`) / `#1A1838` (`card2`) / `rgba(199,203,247,.12)` (`hair` border) | Tracklist/credits/metadata cards, Files-row backgrounds, Export Pack side-panel background |
| Accent (10%) | `linear-gradient(105deg,#818CF8 0%,#D946EF 100%)` (`bg-grad` / `gtext`) | Reserved for: the play/pause transport button fill, the active Master/Instrumental toggle option fill, the "Download ZIP now" primary button fill (Export Pack panel), the upload-progress bar fill |
| Destructive | not used this phase | No destructive action exists in D-01 through D-14's decisions (uploads are additive, no delete/revoke UI) |

**Semantic status colors (existing project tokens, reused — not new):**
| Tone | Value | Usage |
|------|-------|-------|
| Success/emerald | `#34D399` (`emerald-400`) | Uploaded-file confirmation ("Uploaded" label), readiness chip when score ≥80 (`readinessLabel` tone `green`) |
| Warning/amber | `#F59E0B` / `#F4C77B` (`money`/`money2`) | Readiness chip when score is 60–79 (tone `amber`) |
| Danger/rose | `#F43F5E` (`rose-400`) | Readiness chip when score <60 (tone `red`); upload/export error copy |

Accent reserved for: play/pause button, active toggle segment, "Download ZIP now" button, upload progress-bar fill. The "Export pack" topbar button and "Download stems" button are **ghost/secondary** style (`bg-card2`, `border-hairstrong`, no gradient) — matching `playback.html`'s `.btn.ghost` class exactly; do not apply the accent gradient to either.

## Layout & Component Inventory

### Topbar (existing `Topbar` component, extended)
- Title/subtitle: unchanged pattern — `"{Project title}"` / `"{Artist} · {Type label} · {N} tracks · {master status}"` (matches `playback.html`'s `Maya Reyes · EP · 5 tracks · master audio uploaded`).
- **New:** Readiness chip (D-02, placement 1) — inserted between the title block and any existing topbar actions. `<Link href={`/vault/${projectId}`}>`, classes: `inline-flex items-center gap-[7px] rounded-full border px-[11px] py-[5px] text-[12.5px] font-bold whitespace-nowrap` + tone-specific color classes (emerald/amber/rose per `readinessLabel()` tone, matching the existing `.chip.ok/.warn/.bad` convention already used by `VaultProjectCard`'s status chip). Includes a small `h-[7px] w-[7px] rounded-full bg-current` dot, matching `.chip .d`.
- **New:** "Export pack" button — ghost style (`inline-flex items-center gap-[9px] rounded-[10px] border border-hairstrong bg-card2 px-5 py-3 text-[15px] font-bold text-white`, box icon `<path d="M4 4h16v16H4z"/><path d="M9 9h6v6H9z"/>`), positioned as the rightmost topbar element. **Hidden or disabled with the "no-master gate" copy above if no track in the project has a master WAV uploaded** (there is nothing meaningful to export yet).

### Left column — Tracklist + Files (existing card, extended)
- Tracklist: unchanged.
- Files section (existing "Master" row unchanged: label "Master", right-aligned "Uploaded"/"Missing" in emerald/lavdim):
  - **New Instrumental row** — inserted directly below Master. Visible to all viewers if present (right side: "Uploaded", emerald). If absent: visible only to `canManage` (owner) with a "+ Add instrumental" upload affordance (mirrors `TrackList.tsx`'s `AudioSlot` pattern — file input, `accept="audio/*"`, hidden native input behind a styled label pill); **hidden entirely for non-owner viewers** (D-08).
  - **New Stems row** — inserted below Instrumental. If present: label "Stems", right side a "Download stems" icon+text button (triggers a signed-URL download, does not touch the play/pause transport). If absent: visible only to `canManage`, with an inline ⓘ info-button next to the "Stems" label (opens the popover copy above — a simple `title`/tooltip or small popover, no new dependency needed) and a "+ Add stems (ZIP)" upload affordance, `accept=".zip,application/zip,application/x-zip-compressed"`. **Hidden entirely for non-owner viewers** when absent (D-08).
  - **Upload progress:** while an upload is in flight (stems or instrumental), the row's right-side control is replaced with "Uploading… {percent}%" (percent driven by `tus-js-client`'s `onProgress` for stems; a simple indeterminate state is acceptable for the smaller instrumental upload). On failure, the row shows the relevant error copy above in `text-rose-400`, upload affordance remains clickable to retry.
  - **New inline readiness widget (D-02, placement 2):** a single-line link row directly below the Files section, inside the same card, separated by the existing `border-t border-hair pt-4` divider convention already used between "Tracks" and "Files". Copy: "Readiness {score}/100 · {label} →", `text-[13px] font-semibold text-lavdim hover:text-white transition`, links to `/vault/${projectId}`.

### Center column — Player (existing, toggle reworked)
- Cover, title, subtitle, waveform, transport: unchanged.
- **Reworked toggle (D-06):** Only rendered when an Instrumental exists (D-08 — no dead/disabled toggle). Same visual container as today (`rounded-[10px] border border-hairstrong bg-card2 p-1 text-[13px] font-bold`), two segments now labeled "Master" / "Instrumental" instead of "Master" / "Stems". Selecting a segment swaps the `<audio src>` between the master and instrumental signed URLs — a pure client-side source swap, no new backend call.
- **New "Download stems" button:** rendered only when a stems ZIP exists (D-08), positioned directly below the toggle (or below the waveform/transport if no toggle is present, since the two are independent per D-04). Ghost/secondary style, small: `inline-flex items-center gap-2 rounded-[9px] border border-hairstrong bg-card2 px-4 py-2 text-[13px] font-bold text-white hover:opacity-90`, download icon (arrow-into-tray, matches `TrackList.tsx`'s existing master-download icon glyph). Triggers a direct file download via signed URL — separate from play/pause, never disables/replaces the transport controls.

### Right column — Credits & Metadata
- Unchanged.

### Export Pack panel (new — right-side slide-over, matching `ToolSidePanel.tsx`'s established pattern)
- Structure: `fixed inset-0 z-50 flex justify-end` wrapper; backdrop `<button>` (`absolute inset-0 bg-black/60 backdrop-blur-sm`, click-to-close); panel `<aside className="relative flex h-full w-full max-w-md flex-col border-l border-hair bg-[#0a0a0f] shadow-2xl">`.
- Header: panel title "Export pack" (18px bold) + description copy (13.5px, `text-lavdim`) + close button (top-right, existing `aria-label="Close panel"` convention).
- Body: included-items list (only existing files/sheets listed, each row with a small emerald checkmark, matching `playback.html`'s Files-row checkmark glyph) → then the two delivery-choice buttons stacked ("Download ZIP now" primary/gradient full-width, "Get shareable link" ghost full-width, `mt-3` gap between them).
- States: idle (both buttons enabled) → generating ("Preparing your pack…" + spinner, both buttons disabled) → result (download: auto-triggers signed-URL navigation + confirmation copy; link: readonly input + "Copy link" button + 7-day expiry helper text) → error (error copy + a "Try again" button that resets to idle).

### States summary

| State | Visual behavior |
|-------|------------------|
| Viewer (non-owner), no Instrumental/Stems | Toggle absent, "Download stems" absent, Instrumental/Stems Files rows absent entirely — playback room looks identical to today's Master-only experience |
| Owner, no Instrumental/Stems | Files rows show "+ Add instrumental" / "+ Add stems (ZIP)" + ⓘ; no toggle, no "Download stems" yet |
| Owner, uploading stems | Stems row shows "Uploading… {percent}%"; upload affordance temporarily replaced, not removed |
| Owner or viewer, Instrumental present only | Toggle shows Master/Instrumental; no "Download stems" button |
| Owner or viewer, Stems present only | No toggle; "Download stems" button visible below transport |
| Both present | Toggle + "Download stems" both visible, independently of each other |
| Export pack, no master uploaded anywhere in the project | Topbar "Export pack" button disabled/hidden with the no-master-gate copy (tooltip or disabled state — executor's choice, must not silently no-op) |
| Export pack, generating | Panel buttons disabled, spinner + "Preparing your pack…" |
| Export pack, download chosen | Signed URL (5 min TTL) auto-triggers browser download; confirmation copy shown |
| Export pack, link chosen | Signed URL (7 day TTL) shown in a readonly field with Copy button; helper text states the expiry |
| Export pack, generation fails | Error copy + "Try again" resets to idle state |
