<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → export route | The playback room POSTs a delivery mode; the route must verify ownership before assembling anything |
| export route → Supabase Storage (service client) | Reads the owner's source files, writes the assembled pack, mints the signed URL — all after owner verification |
| signed export URL → recipient (unauthenticated) | The shareable link exposes actual master/stems/instrumental files to anyone holding it, for 7 days — materially more sensitive than Phase 9's stream-only public link |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-14-12 | Elevation of Privilege | export route invoked for another user's project | high | mitigate | The route fetches project + tracks with `.eq('user_id', user.id)` and returns 404 before any Storage read or URL minting (ASVS V4, mirrors the embed route). Verified by node grep + the two-session manual check (VALIDATION Manual-Only). |
| T-14-13 | Information Disclosure | forever-live / mis-scoped export link | high | mitigate | The link is a Supabase `createSignedUrl` with an explicit TTL in SECONDS (`60*60*24*7` share / `60*5` download); Storage enforces expiry server-side, so no manual revocation and no `expires_at` bookkeeping (D-12, RESEARCH §Don't Hand-Roll). The route re-derives a fresh signed URL per request against a stable path rather than reusing an indefinitely-lived URL (RESEARCH §Security Domain). Unit-mismatch guarded by the node verify requiring the exact `60*60*24*7` literal. |
| T-14-14 | Information Disclosure | conflating export link with Phase 9's public `/r/[id]` player | high | mitigate | The export link uses NO `is_public` flag and NO public route — it is a per-request, owner-generated, time-boxed signed URL only (RESEARCH §Pitfall 5). No `vault_projects` "pack is public" boolean is added. |
| T-14-15 | Denial of Service | assembly exceeding the Hobby 10s maxDuration | medium | mitigate | `maxDuration=10` set honestly; `zlib:{level:0}` (store) minimizes CPU on already-compressed inputs; the route uploads-then-signs rather than streaming the archive out (so recipient download time is off the function budget). Residual risk (very large albums) is a deployment finding surfaced by HOBBY-2 with a Vercel-Pro-upgrade remedy — not mitigated by speculative job-queue infra (RESEARCH Pitfall 3). |
</threat_model>
