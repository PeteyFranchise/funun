<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser tab → presence-global channel | Presence is broadcast to all signed-in members (D-23); the user-scoped key prevents a tab from impersonating a different user's presence entry. |
| client → /api/dm/threads?unread=true | The badge count is server-computed; the client only displays it and cannot forge a lower/higher unread number that affects anyone else. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-11-12 | Spoofing | multi-tab ghost / impersonated presence | medium | mitigate | User-scoped presence key (`key: userId`) coalesces tabs into one entry; no randomUUID key (Pitfall 2); heartbeat writes only the caller's own last_seen_at |
| T-11-13 | Denial of Service | Realtime concurrent-connection exhaustion | medium | mitigate | ONE presence channel per user mounted once at layout root (Pitfall 1/4); no per-surface presence channels; connection-budget note recorded in SUMMARY per STATE.md pending todo |
| T-11-14 | Tampering | client-inflated unread badge | low | mitigate | Badge set only from the fresh GET count, never client-incremented (D-07); a forged local number affects only the attacker's own display |
| T-11-SC | Tampering | npm installs (lucide-react temptation) | high | mitigate | No packages installed; all icons are inline SVG (existing pattern). If any install is attempted, halt and add a blocking-human legitimacy checkpoint |
</threat_model>
