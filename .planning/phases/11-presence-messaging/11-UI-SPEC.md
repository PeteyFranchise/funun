## Typography

Exactly 4 font sizes, exactly 2 weights. No exceptions.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Thread-list preview text, message bubble body, inbox search input, message timestamps inside bubbles (opacity-60 for de-emphasis) |
| Label | 12px | 700 (bold) | 1.4 | Presence status ("Active now"), date dividers, unread badge counts on topbar icon, request/section labels, rate-limit budget hint ("7 requests left"), thread-list timestamp, section headers (uppercase letter-spacing `.18em` for section headers only) |
| Heading | 15px | 700 (bold) | 1.2 | Thread row name, docked widget header name, conversation header name |
| Display | 27px | 700 (bold) | 1.1 | `/messages` page title (matches existing topbar `h1` pattern) |

**Consolidation rationale (checker fix applied 2026-07-13):**
- Unread badge (was 10px/800) â†’ 12px label slot, weight 700. Badge is small enough that the container clips the glyph naturally.
- Bubble timestamps (was 10px/400) â†’ 14px body slot, opacity-60. Inline with message body; de-emphasis handled by opacity alone.
- Thread-list timestamp and section headers (were 11px/600â€“700) â†’ 12px label slot, weight 700. 12px at 700 is visually equivalent for micro-labels at this scale.
- Weights collapsed: 400 (regular) and 700 (bold) only. Semibold (600) merged into bold (700); extrabold (800) merged into bold (700).

## Interaction Contracts

### `/messages` two-pane layout

- **Breakpoint:** `lg` (1024px). Below 1024px: single-pane (thread list â†’ tap â†’ full-screen conversation, no floating widget). At or above 1024px: two-pane + docked widget available.
- **Thread list width:** 320px fixed, `flex-none`. Conversation pane: `flex-1`.
- **Active thread:** highlighted `bg-card2` with `border-l-[3px] border-brandindigo` left edge.
- **Thread navigation:** clicking a thread sets it active, opens `ConversationView` on the right, auto-marks read (D-06), fires `POST /api/dm/read/{threadId}`.
- **Pending/requests threads:** rendered in a separate "Requests" section below the normal thread list, visually separated by a section label. A `chip.info` badge next to the "Requests" label shows the pending count.

### Docked widget (DockedWidget)

- Mounted in `app/(artist)/layout.tsx` at layout root â€” persists across SPA navigation.
- Opens when: user clicks "pop out" icon inside `ConversationView` on `/messages`.
- Closes when: user clicks close (chevron-down or âś•). State: `useState<string | null>(null)` in layout.
- At or above `lg` breakpoint only (hidden on mobile with `hidden lg:block`).
- Position: `fixed bottom-0 right-8 z-50` â€” carries from existing `DmWidget.tsx` positioning.
- Width: 336px (`.pf-dm` design spec).
- Auto-scrolls to bottom on new message (mirrors `DmWidget.tsx` `listRef.current?.scrollTo`).
- No overlapping: only one conversation docked at a time (future phase can expand to multi-dock; single-dock is Phase 11 scope).

### Message composer

- `Enter` key to send (matches existing `DmWidget.tsx` `onKeyDown` pattern).
- `maxLength={4000}` (inherits from existing `DmWidget.tsx`).
- Send button: **icon-only** (paper-plane SVG, no visible label). Accessible name: `aria-label="Send"`. Disabled when input is empty or `busy` (optimistic send in flight).
- For non-connections: `RequestsBudgetHint` shown below input when budget > 0; `RateLimitWall` replaces composer entirely when budget = 0.
- Pending thread (sender side, `status='pending'`): composer visible but shows stacked-message hint when message count â‰Ą 1. Composer disabled at 3 messages.

### Optimistic send

- Insert temporary message immediately into the conversation view (same pattern as `DmWidget.tsx` â€” `tmpId = 'tmp-${Date.now()}'`).
- On API error: revert (remove tmp row).
- On API success: swap tmp row for the real row returned by the API.

### Auto-read

- Opening a thread (via thread list click, or docked widget open): immediately fire `POST /api/dm/read/{threadId}` (best-effort, no await, no error surfacing to user).
- No explicit "mark read" control.

### Realtime / polling

- Thread message delivery: Supabase `postgres_changes` INSERT on `dm_messages` for the active thread + 20s poll fallback (carries from `DmWidget.tsx`).
- Unread badge on `MessagesIcon`: Realtime INSERT on `dm_messages` â†’ fresh `GET /api/dm/threads?unread=true` count. Never client-incremented.
- Presence: Supabase Realtime Presence channel `presence-global` with `config: { presence: { key: userId } }`. One channel per user; `visibilitychange` re-track; explicit `removeChannel` on unmount.
