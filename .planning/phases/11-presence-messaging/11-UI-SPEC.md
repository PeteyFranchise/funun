## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), dot margins, badge offset |
| sm | 8px | Chip padding, inline element gaps (`gap-2`) |
| md | 16px | Widget body padding, thread-row padding, composer padding |
| lg | 24px | Section gap in `/messages` two-pane layout |
| xl | 32px | Page horizontal padding, docked widget `right-8` offset |
| 2xl | 48px | `/messages` page top padding |
| 3xl | 64px | Major layout breaks (reserved; rarely used this phase) |

Exceptions:
- Docked widget width: 336px (carried from `.pf-dm` CSS — inherited design value, not a spacing token)
- Docked widget avatar: 36×36px (`h-9 w-9`) — design-spec value preserved from existing `DmWidget.tsx`
- Presence dot: 6×6px inside widget header; 7×7px inside `.pf-avatar .live` pill and thread-list avatars
- Topbar icon button: 42×42px (`h-[42px] w-[42px]`) — mirrors `NotificationBell` exactly (D-02 requirement: messages icon is a peer of the bell)
- Touch-safe minimum: all interactive rows ≥ 44px height on mobile

## Presence Dot States

| State | Visual |
|-------|--------|
| Active now (Realtime Presence) | 6–7px filled circle `bg-emerald-400` (green) |
| Active X ago / today / this week (`last_seen_at` bucket) | No dot — status text only in DM header; no dot in thread list for offline users |

**Presence dot placement rules (D-22):**
- Profile avatar: `.live` pill (emerald dot + "Online" text) — absolute bottom-right of 168px avatar
- Thread list row: 6px dot overlaid bottom-right of 40px avatar
- Two-pane conversation header: 6px dot + "Active now" text (emerald) or no dot + "Active Xm ago" text (lavdim)
- Docked widget header: identical to conversation header (the `dh .st` pattern from `.pf-dm`)
- Notification panel avatars: no presence dot (D-22 exclusion — Phase 12 deferred)
- Wall post avatars: no presence dot (D-22 exclusion)

## Interaction Contracts

### `/messages` two-pane layout

- **Breakpoint:** `lg` (1024px). Below 1024px: single-pane (thread list → tap → full-screen conversation, no floating widget). At or above 1024px: two-pane + docked widget available.
- **Thread list width:** 320px fixed, `flex-none`. Conversation pane: `flex-1`.
- **Active thread:** highlighted `bg-card2` with `border-l-[3px] border-brandindigo` left edge.
- **Thread navigation:** clicking a thread sets it active, opens `ConversationView` on the right, auto-marks read (D-06), fires `POST /api/dm/read/{threadId}`.
- **Pending/requests threads:** rendered in a separate "Requests" section below the normal thread list, visually separated by a section label. A `chip.info` badge next to the "Requests" label shows the pending count.

### Docked widget (DockedWidget)

- Mounted in `app/(artist)/layout.tsx` at layout root — persists across SPA navigation.
- Opens when: user clicks "pop out" icon inside `ConversationView` on `/messages`.
- Closes when: user clicks close (chevron-down or ✕). State: `useState<string | null>(null)` in layout.
- At or above `lg` breakpoint only (hidden on mobile with `hidden lg:block`).
- Position: `fixed bottom-0 right-8 z-50` — carries from existing `DmWidget.tsx` positioning.
- Width: 336px (`.pf-dm` design spec).
- Auto-scrolls to bottom on new message (mirrors `DmWidget.tsx` `listRef.current?.scrollTo`).
- No overlapping: only one conversation docked at a time (future phase can expand to multi-dock; single-dock is Phase 11 scope).

### Message composer

- `Enter` key to send (matches existing `DmWidget.tsx` `onKeyDown` pattern).
- `maxLength={4000}` (inherits from existing `DmWidget.tsx`).
- Send button: disabled when input is empty or `busy` (optimistic send in flight).
- For non-connections: `RequestsBudgetHint` shown below input when budget > 0; `RateLimitWall` replaces composer entirely when budget = 0.
- Pending thread (sender side, `status='pending'`): composer visible but shows stacked-message hint when message count ≥ 1. Composer disabled at 3 messages.

### Optimistic send

- Insert temporary message immediately into the conversation view (same pattern as `DmWidget.tsx` — `tmpId = 'tmp-${Date.now()}'`).
- On API error: revert (remove tmp row).
- On API success: swap tmp row for the real row returned by the API.

### Auto-read

- Opening a thread (via thread list click, or docked widget open): immediately fire `POST /api/dm/read/{threadId}` (best-effort, no await, no error surfacing to user).
- No explicit "mark read" control.

### Realtime / polling

- Thread message delivery: Supabase `postgres_changes` INSERT on `dm_messages` for the active thread + 20s poll fallback (carries from `DmWidget.tsx`).
- Unread badge on `MessagesIcon`: Realtime INSERT on `dm_messages` → fresh `GET /api/dm/threads?unread=true` count. Never client-incremented.
- Presence: Supabase Realtime Presence channel `presence-global` with `config: { presence: { key: userId } }`. One channel per user; `visibilitychange` re-track; explicit `removeChannel` on unmount.
