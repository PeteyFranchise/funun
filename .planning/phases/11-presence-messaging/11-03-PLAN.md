<objective>
Build the Phase 11 DM API layer: modify `POST /api/dm/send` to add the connection gate + rate limit + request flow, and create six net-new routes — `GET /api/dm/threads` (thread list + unread), `POST /api/dm/read/[threadId]` (auto-read marker), the three request-lifecycle routes (`accept`/`decline`/`block`), and `POST /api/presence/heartbeat` (throttled last_seen_at write). Output: one modified route, six new routes, and a send-gate unit test.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /api/dm/send | The single choke point for "who may message whom, how often." The connection gate, rate limit, and stacked-cap all live here — not in the browser. |
| client → /api/dm/request/* | Recipient-only state transitions; the requester must not be able to accept/decline their own request. |
| client → /api/presence/heartbeat | Members write only their OWN last_seen_at, and only via the throttled service-role update — no forging another member's presence. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-11-06 | Elevation of Privilege | connection-gate bypass via direct PostgREST on dm_messages | high | mitigate | The gate is enforced in /api/dm/send (isConnected + rate limit before insert); dm_threads INSERT already carries no_block(); all DM creation funnels through this one route (RESEARCH V4 control) |
| T-11-07 | Denial of Service | message-request spam / notification flooding | high | mitigate | Rolling 7-day rate limit (10/30) returns 429; pending-stack cap of 3; message_request notification fires once per cold thread only, not per stacked message (D-16/D-18) |
| T-11-08 | Spoofing | requester self-accepting their own message request | high | mitigate | accept route rejects when `updated.requester_id === user.id` (403) and the `.eq('status','pending')` + zero-row→404 transition guard |
| T-11-09 | Information Disclosure | leaking block relationships via the request endpoints | medium | mitigate | block is an INSERT of the caller's own row (blocks_insert_own, session client); decline is silent (D-11) so a declined sender learns nothing; no endpoint returns another user's blocklist |
| T-11-10 | Tampering | forging presence "Active now" for another member | high | mitigate | heartbeat updates only `.eq('id', user.id)` via service client; no last_seen_at UPDATE grant exists for authenticated (migration 054) |
| T-11-11 | Tampering | XSS via message body in previews/notifications | medium | mitigate | message_request preview is truncated plain text stored as-is; React renders it as text (Plans 05/06), and createNotification's esc() escapes email HTML (existing) |
| T-11-SC | Tampering | npm installs | high | mitigate | No new packages introduced; if any install is attempted, halt and add a blocking-human legitimacy checkpoint |
