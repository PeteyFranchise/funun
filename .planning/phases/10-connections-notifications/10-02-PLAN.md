<objective>
Ship the single additive migration this phase needs (`044_connections_note.sql`) and push it live. Three changes, all inside one migration: (1) add the `connections.note` column with a 200-char CHECK (D-04); (2) close the `no_block()` wiring gap on `connections` INSERT that migration 038 left open (Pitfall 2 — inert today, but must be live before Phase 13 populates blocks); (3) add the `SECURITY DEFINER` auto-follow-seed trigger so accepting a connection creates both follow rows atomically — which the accepting user's own RLS session physically cannot do (Pitfall 1, the single highest-value finding in this phase's research).

A DB trigger is the correct, RLS-safe, defense-in-depth mechanism (fires regardless of which path performs the status transition). The `no_block()` gap is a five-minute additive fix that avoids a Phase 13 retrofit migration. This is a schema-modifying plan, so it carries the mandatory [BLOCKING] schema push.
