# Phase 10 ‚Äî UI Design Contract: Connections & Notifications

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Read this first:** This phase has almost zero visual precedent to recreate ‚Äî RESEARCH.md's Pitfall 4 confirms no topbar exists anywhere in the authenticated app shell today (`app/(artist)/layout.tsx` renders only `<ArtistNav>` beside `{children}`, no header row), and the design bundle's `.pf-iconbtn`/`.dotn` bell lives only on the public `/u/[handle]` page's own self-contained `.pf-top` bar, whose own CSS says "overrides the app shell." This spec designs three genuinely new surfaces: (1) a slim authenticated-app header row to host the bell, (2) the `NotificationBell` + unread-count badge + dropdown panel (D-08‚ÄìD-13), and (3) the `ConnectButton` three-state control + inline Accept/Decline + optional note composer (D-01‚ÄìD-04). It reuses the existing bespoke design system verbatim ‚Äî no new tokens, no new component library.

Governing source remains the exact px values already used throughout `app.css`/`user-profile.html` and implemented in `ProfileView.tsx`/`DmWidget.tsx`/`ProfileMoreMenu.tsx` ‚Äî declare the nominal scale for new spacing decisions, do not force-fit locked measurements onto it.

| sm | 8‚Äì9px | Compact gaps (button icon gap, panel row internal gap) |

| lg | 22‚Äì24px | Button horizontal padding, note composer padding |

**New surface measurements (locked, reuse verbatim):**

- **Header row** (new, `app/(artist)/layout.tsx` content column, above `{children}`): `16px 24px` padding, `1px` bottom border (`--border`), sticky `top:0`, `rgba(10,10,15,.72)` bg + `backdrop-filter: blur(20px)` ‚Äî same recipe as `.pf-top`, but with no logo/nav/search (those live in `ArtistNav`/are out of this phase's scope); content is right-aligned, bell only.
- **Bell button:** 42√ó42px, 11px radius ‚Äî reuse `.pf-iconbtn` exactly (`card` bg, `1px border`, `lav` icon, 20√ó20px SVG, 1.7 stroke).
- **Unread badge (count, not a bare dot ‚Äî see Color/Copywriting):** min 16√ó16px pill, 999px radius, positioned `-4px` top/right of the bell button (adjusted from `.dotn`'s 8px/9px inset since this badge carries a number), `2px` `card`-colored border matching `.dotn`'s ring convention.
- **Dropdown panel:** anchored `absolute right-0 top-full mt-2`, width **380px**, `12px` radius, `1px` `hair` border, `card` bg, `0 12px 30px -10px rgba(0,0,0,.5)` shadow ‚Äî same shadow/border recipe as `ProfileMoreMenu`, wider to fit two-line rows + inline action buttons.
- **Panel header:** `16px 18px` padding, `1px` bottom `hair` border.
- **Panel row:** `14px 18px` padding, min-height 64px (84px when inline Accept/Decline buttons are present), `1px` bottom `hair` border between rows (omit on last row).
- **Row icon/avatar tile:** 36√ó36px, rounded (`10px` for icon tiles matching `.pf-rolechip` radius family, `50%` for actual avatar images).
- **Inline Accept/Decline buttons (panel row + profile ConnectButton addressee view):** 36px height, `9px` radius, `8px 14px` padding, `13px/700` label ‚Äî a compact sibling of the `.pf-btn` family, not the full 42px/`13px 22px` profile-action size.
- **Connect-note composer popover:** anchored below the Connect button, width **320px**, `12px` radius, `1px` `hair` border, `card` bg, same shadow as the dropdown panel; `16px` internal padding.
- **List max-height (scroll-to-load, D-11):** 420px, `overflow-y: auto`.

Exceptions: none beyond the above ‚Äî all new measurements are declared here, not left to "consider ~16px" ambiguity.

Accent reserved for: the **Connect** button specifically (not Follow ‚Äî see rationale), Accept buttons (inline panel + profile), the connection-request icon tile tint, and the unread-row indicator dot. Never apply the gradient to Follow, Message, Decline, or informational panel rows.

**Locked visual-weight decision (closes CONTEXT.md's "Follow and Connect should read as clearly different weights, not near-duplicates" note):** **Connect is the primary/gradient CTA; Follow moves to the ghost/secondary treatment** (`border-hairstrong bg-card text-white`, same visual family it already has for the "Following" toggle state). This intentionally flips today's `user-profile.html` mockup, where Follow was the only relationship action and got the primary treatment ‚Äî now that Connect exists as the deliberate, mutual, professional-network action this milestone's core value centers on ("real relationships... connect with producers, supervisors, A&R, and execs"), it earns the higher-contrast slot. Follow stays zero-friction but visually quieter; Message stays ghost/secondary as it already is via `DmWidget`.

## New Surface Specifications

### 1. Authenticated header row (net-new ‚Äî Pitfall 4)

- Add a slim sticky header inside `app/(artist)/layout.tsx`'s content column, directly above `{children}`: `<div className="sticky top-0 z-40 flex items-center justify-end border-b border-hair bg-[rgba(10,10,15,.72)] px-6 py-4 backdrop-blur-[20px]">`.
- Content: right-aligned, `<NotificationBell />` only. No logo/search/nav ‚Äî those already live in `ArtistNav`'s left sidebar and are out of this phase's scope (search bar is Phase 12/DISCOVER-01).
- Mounted exactly once, at the layout level, so it renders on every authenticated route ‚Äî this is what satisfies D-13's "global, not gated to dropdown-open" requirement architecturally (RESEARCH's recommendation A3).

### 2. `NotificationBell` ‚Äî icon + unread badge (NOTIF-02, D-12, D-13)

- Button: reuse `.pf-iconbtn` classes exactly (42√ó42px, 11px radius, `card` bg, `1px border` `hair`, `lav` icon color). Bell glyph: the actual bell path from `user-profile.html`'s second `.pf-iconbtn` (`M18 8a6 6 0 0 0-12 0c0 7-3 9-3 9h18s-3-2-3-9` + clapper `M13.7 21a2 2 0 0 1-3.4 0`) ‚Äî **not** the chat-bubble path used for the first icon in that mockup (that one is the messages icon, out of scope ‚Äî Phase 11 owns the DM widget's own affordance).
- Badge: when unread count > 0, render a numbered pill (see Spacing: 16√ó16px min, fuchsia fill, `2px` `card`-colored ring) with `10px/800` white tabular-nums text, capped display at `9+` for counts ‚â• 10. When count is 0, render nothing ‚Äî no empty dot, no "0".
- Data: unread count comes from a fresh `COUNT` query (`GET /api/notifications?unread=true`, head-count style) on every poll/realtime tick ‚Äî never derived from client-side increment/decrement (RESEARCH anti-pattern guard, STATE.md's explicit prior decision).
- Realtime: subscribes to `postgres_changes` INSERT on `notifications` filtered to the current user, **globally** (not gated to the dropdown being open, unlike `DmWidget`) ‚Äî with a stable per-user channel name (`notifications-${userId}`) and proper `removeChannel` cleanup on unmount (RESEARCH Pitfall 5). ~20‚Äì30s poll as reconcile fallback, mirroring `DmWidget.tsx`'s existing interval pattern.
- Click: toggles the dropdown panel open/closed. Opening the panel does **not** clear the badge (D-09) ‚Äî only the explicit "Mark all read" click does.

### 3. `NotificationPanel` ‚Äî dropdown (NOTIF-03, D-08‚ÄìD-11)

- Container: anchored to the bell (`absolute right-0 top-full mt-2`, 380px wide, 12px radius, `card` bg, `1px hair` border, `0 12px 30px -10px rgba(0,0,0,.5)` shadow) ‚Äî same click-outside-to-close pattern as `ProfileMoreMenu.tsx` (`mousedown` listener on a wrapping ref).
- Header row: `Notifications` (16px/800 white, left) + `Mark all read` (13px/600 indigo, right; dimmed/disabled at unread=0).
- List: max-height 420px, `overflow-y: auto`. Each row:
  - Left: 36px icon/avatar tile ‚Äî actor's avatar image if the notification carries `actor_avatar_url`, else a type-icon tile (see item 5, icon catalog) on `card2` bg (or a gradient-tinted tile for `connection_request`, see Color contract).
  - Middle: title (14px/600 white, 1 line, truncate) + body/note (13px/500 lavdim, 2-line clamp, omitted if empty) + timestamp (12px lavdim, reuse the `timeAgo()` helper pattern from `components/profile/Wall.tsx` verbatim ‚Äî no `date-fns`).
  - Unread state: row bg `card2` + a small 8px fuchsia dot at the row's left inset (4px from edge). Read state: row bg transparent, no dot.
  - Inline actions (connection_request only, D-10): a row of two compact buttons directly beneath the body text ‚Äî `Accept` (compact gradient, 36px height) and `Decline` (compact ghost). Clicking either calls the connect-respond API and updates the row in place (removes the inline actions, updates title to reflect the new state ‚Äî e.g. swaps to a plain "You accepted {actorName}'s request" line) without closing the panel or navigating away.
  - Row click (anywhere except the inline action buttons) navigates to the notification's `link` target (see item 6, deep-link catalog); inline-button clicks call `event.stopPropagation()` so they never trigger row navigation.
- Empty state: centered, ~40px vertical padding ‚Äî a 40px bell-icon tile on `card2` bg, then the empty-state heading/body from the Copywriting Contract.
- Pagination (D-11): loads 20 most recent on open; an `IntersectionObserver` sentinel at the list's bottom auto-fetches the next page (`created_at`-cursor pagination per RESEARCH's Open Question recommendation ‚Äî not offset-based) while scrolling, showing the `Loading‚Ä¶` label from the Copywriting Contract during the fetch. No older-notifications cutoff ‚Äî the DB remains fully queryable.
- Error state: if the initial fetch fails, show the panel's error copy (Copywriting Contract) in place of the list, with the `Retry` action re-triggering the fetch.

### 4. `ConnectButton` ‚Äî three-state control (CONNECT-02, D-01‚ÄìD-04)

Button order on the profile action row is locked: **Connect, Follow, Message** (D-01) ‚Äî reuses the exact `42px`/`13px 22px`/`11px radius`/`15px 700` sizing already established by `FollowButton.tsx`/`ProfileView.tsx`'s action row, just re-skinned per the Color contract's visual-weight decision.

- **State: `none`** (no connection row exists, or a prior request reached a terminal state ‚Äî declined/withdrawn) ‚Üí renders `Connect`, primary-gradient styling (`bg-grad text-white shadow-cta`), icon: user-plus glyph. Click opens the note composer popover (see item 5).
- **State: `pending` (viewer is requester)** ‚Üí renders `Pending`, ghost styling (`border-hairstrong bg-card text-white`). On hover/focus, label + icon swap to `Withdraw` (icon: x-circle), no color change (stays neutral, not rose ‚Äî matches the "no destructive styling for reversible actions" contract). Click immediately calls the withdraw transition (`connections.status='withdrawn'`), optimistic UI reverts the button to `Connect` ‚Äî no confirmation dialog.
- **State: `pending` (viewer is addressee)** ‚Üí the whole Connect button is **replaced** by two inline buttons, `Accept` (compact gradient) and `Decline` (compact ghost), same compact 36px sizing as the panel's inline actions (item 3). If the request carries a `note`, render it directly above the two buttons as a small quoted callout: `13px/500` lavdim text on a `card2` background pill, max-width matching the action row, no quote-mark glyph (plain text is sufficient at this size).
- **State: `connected`** ‚Üí renders `Connected`, ghost styling, icon: check glyph, **non-interactive** (no `onClick`, default cursor) ‚Äî this is an accurate terminal status this phase, not a stub; disconnect lands in Phase 13 per D-07 and gets its own affordance then.

### 5. Connect-note composer popover (D-04)

- Trigger: clicking `Connect` in the `none` state opens this popover, anchored below the button (see Spacing: 320px wide, same shadow/radius family as the notification panel).
- Content: a `<textarea>` (placeholder + char counter from the Copywriting Contract), `maxLength={200}`, 15px/500 body text, `card2` bg, `1px hair` border, `10px` radius, ~3 rows tall. Below it: `Send request` (compact primary gradient button) + `Cancel` (compact ghost/text button), right-aligned.
- Submit behavior: POSTs the connect-request with `note` set to the trimmed textarea value (or `null` if empty ‚Äî an empty note is a fully valid, common case, not an error). On success, the popover closes and the `ConnectButton` transitions to the `pending` (requester) state. On failure, the popover stays open with an inline error line beneath the buttons (reuse the panel's warn-toned error text convention ‚Äî amber, not rose).
- Click-outside-to-close: same `mousedown`-listener pattern as `ProfileMoreMenu.tsx`.

Locked catalog closing both open discretion points from CONTEXT.md (icon treatment + deep-link targets). Each entry: hand-rolled SVG (1.7 stroke, round caps, matching `components/nav/icons.tsx` conventions), icon tile treatment, and `link` value.

| Type | Icon (new, hand-drawn to match existing convention) | Tile treatment | `link` target |
|------|------|----------------|----------------|
| `new_follower` | user-plus (person outline + `+`) | plain `card2` tile, `lav` icon | `/u/{actorHandle}` |
| `connection_request` | link (two overlapping rings) | **gradient-tinted tile** (`bg-gradient-to-br from-brandindigo/20 to-brandfuchsia/20`, indigo icon) ‚Äî visually flags "needs your action" | `/u/{actorHandle}` (row click only; inline Accept/Decline handle the action without navigating) |
| `connection_accepted` | check (checkmark) | plain `card2` tile | `/u/{actorHandle}` |
| `wall_post` | message-square | plain `card2` tile | `/u/{ownHandle}#wall` (this phase adds `id="wall"` to the existing `<Wall>` section wrapper in `ProfileView.tsx` to make the anchor real) |
