  <name>Task 1: Extend EsignProvider contract for DocuSeal + pure webhook verification</name>
  <files>lib/esign/provider.ts, lib/esign/webhook.ts, lib/esign/webhook.test.ts</files>
  <behavior>
    verifyDocusealSignature(rawBody, header, secret, nowMs):
    - Valid HMAC-SHA256 over the raw body with the shared secret, timestamp within 5 minutes → returns true.
    - Tampered body (signature computed over different bytes) → false.
    - Stale timestamp (older than 5 minutes) → false.
    - Missing/malformed header → false.
    - Comparison is constant-time (uses crypto.timingSafeEqual), never a plain === on the hex digests.
    parseDocusealEvent(payload):
    - Maps a submission.completed payload to { type: 'all_signed', requestId, ... }.
    - Maps a form.completed / per-signer completion to { type: 'signed', requestId, signerEmail }.
    - Maps form.declined to { type: 'declined' }; an unknown event maps to { type: 'other' }.
    readEsignState round-trip:
    - A document_data.esign with provider 'docuseal' reads back as provider 'docuseal' (not coerced to dropbox_sign).
  </behavior>
  <action>
    In lib/esign/provider.ts, add 'docuseal' to the EsignState.provider union and update readEsignState's provider-narrowing branch so a persisted 'docuseal' value round-trips instead of falling through to the legacy default. Change nothing else about the interface contract — the whole point of the seam is that callers (16-09's SignWell adapter reuses this) are unaffected (ESIGN-01, D-18b dual-provider).

    Create lib/esign/webhook.ts as a pure module (Node built-in crypto only, no fetch, no Supabase) exporting verifyDocusealSignature and parseDocusealEvent per the behavior block. Follow the documented DocuSeal X-Docuseal-Signature scheme (timestamp-plus-HMAC-SHA256 over the raw body, 5-minute staleness window) recorded in RESEARCH Security Domain; do not hand-roll the hash and always compare with crypto.timingSafeEqual (RESEARCH Don't Hand-Roll; V6 Cryptography). Keep the exact provider event-name mapping in a single lookup so 17-07's route only wires I/O around it. Treat the event names as the researched set but tolerate unknown types by returning 'other' rather than throwing (RESEARCH Assumption A2 — names confirmed at the provider trial).

    Write webhook.test.ts to the behavior block first (RED), constructing valid and invalid signatures with Node's own crypto against a fixture secret — no live DocuSeal call — then implement to green.
