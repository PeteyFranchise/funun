<objective>
Fix a SHIPPED correctness bug in Funūn's PDF layer (ESIGN-15, P17-08): `@react-pdf/renderer`'s standard-14 fonts use WinAnsi encoding, so any character outside Latin-1 is destroyed at render time. `ć` (U+0107) is silently DROPPED — `Nikola Jokić` becomes `Nikola Joki` — and `ū` (U+016B) mangles, which is why the brand string in the footer of every credits sheet, metadata sheet, and split sheet ever generated reads `Funkn` instead of `Funūn`. Bundle Noto Sans (SIL Open Font License), register it once in a shared module, and switch all three renderers onto it. Also fix the dangling `Prepared by` label when no initiator name is present.

Purpose: This is not cosmetic. Funūn is a rights-documentation product, and the split sheet is the instrument that governs a collaborator's royalty share. Rendering a party's legal name incorrectly on the document they sign is a defect in the core artifact, and it breaks Polish, Czech, Croatian, Serbian, Turkish, Baltic, Vietnamese, and all non-Latin names. It ships BEFORE the legal-grade rebuild (17-09) and before any live envelope mints (17-06), so no real artist ever signs a document with a corrupted name on it. Credential-free, no dependency on any other Phase 17 plan — this can execute immediately.

Scope note (shipped-bug disclosure): every credits sheet and metadata sheet previously generated by Funūn carries the corrupted brand string, and any collaborator whose name contains a non-Latin-1 character has had their name corrupted on a generated document. Record this in the plan summary; remediation of already-generated artifacts is out of scope here (they regenerate from live data on next export).

Output: Vendored, license-compliant Noto Sans assets with recorded provenance; a single `registerFunuunPdfFonts()` module; all three renderers on the registered family; and a Unicode regression suite that proves exact strings survive a real render.
</objective>

| Boundary | Description |
|----------|-------------|
| upstream font release → repository binary | A binary artifact fetched from the internet is committed into the repo and executed by the PDF layer on every render. |
| server DB state → rendered legal document | Party names flow from the database into the instrument the party signs; any lossy transformation in between is a defect in the legal record. |
| local filesystem → deployed serverless bundle | Fonts resolved by absolute path at runtime must be present in the deployment output, not merely in the git checkout. |
