## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authenticated session → esign tables | Signing-state rows must be server-owned; no client may INSERT/UPDATE/DELETE. |
| party session → other parties' rows | A party may read only its own signer row and the sheets it is named on. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |

| T-17-06 | Tampering | client writing forged envelope/signer state | high | mitigate | RLS enabled + INSERT/UPDATE/DELETE revoked from authenticated/anon on both tables (migrations 040/056/058 doctrine); writes only via service-role routes. |
| T-17-07 | Information Disclosure | a party reading another party's or another initiator's signing state | high | mitigate | SELECT policies scoped to initiator-of-sheet and own-row, mirroring migration 018's split_sheet_parties policy pair. |
| T-17-08 | Tampering | readiness score divergence between SQL trigger and TS twin | medium | mitigate | Single shared tier map (17-01) + a shared-fixture parity test asserting both derivations agree (RESEARCH Pitfall 3). |
