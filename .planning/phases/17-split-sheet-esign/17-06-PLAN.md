<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| DocuSeal API → server | Server-only X-Auth-Token calls; the key must never reach the browser. |
| browser → embedded iframe | The signer interacts with a per-signer-scoped embed; only a scoped src crosses, never the API key. |
| client → mint trigger | A privileged, cost-bearing action that must run server-side after cap + ownership checks. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-17-15 | Information Disclosure | DOCUSEAL_API_KEY exposure | high | mitigate | Server-only env var; the client embed receives only a per-signer scoped src/token; the mint call is server-owned (RESEARCH V13). |
| T-17-16 | Elevation of Privilege (cost) | scripting mint calls to exceed the ~10/mo cap | high | mitigate | Server-side checkMonthlyCap inside the mint route before any DocuSeal call, counting the calendar month's esign_envelopes (AM-2). |
| T-17-17 | Tampering | arbitrary-PDF bypass of AM-2 template-only | high | mitigate | The mint route only ever posts 17-03's server-rendered buffer to /templates/pdf; no client file bytes reach the template API. |
| T-17-18 | Repudiation | voided envelope losing audit history | medium | mitigate | Void marks the row 'voided' with voided_at (never deletes); each re-mint is a new row (P17-02). |
| T-17-19 | Elevation of Privilege | signer-token confusion across parties | medium | mitigate | The embed src is minted per esign_envelope_signers row and is never derived from the Funūn approval token. |
| T-17-SC | Tampering | npm install @docuseal/react | medium | mitigate | Package verified at the blocking checkpoint (npmjs.com, MIT, no postinstall, verdict OK per RESEARCH audit); only the MIT client embed SDK is added — no server SDK. |
</threat_model>
