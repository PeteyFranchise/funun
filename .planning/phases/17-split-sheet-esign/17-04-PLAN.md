<objective>
Fix the approve→sign convergence on the existing public token surface without touching DocuSeal: split the /approve/[token] gating so the same durable link now hosts the post-approval signing step (RESEARCH Pitfall 1 / gap fix 1, P17-01), render the phase-correct UI branch (approve/counter, sign, waiting, or genuinely-expired), stamp the page-visit signal that powers the viewed-no-action nudge (P17-04), and widen the PATCH status allowlist so the new statuses are accepted (ESIGN-04, ESIGN-06, ESIGN-09).

Purpose: The party's token IS their durable identity link; forking a second /sign route would mean a second link to explain to studio collaborators. Reusing it requires the gating rewrite the current code actively breaks on (any non-pending status renders "expired"). This plan builds the sign-phase shell and mounts a placeholder region; 17-06 drops the live embed in behind the provider checkpoint. Credential-free.
Output: The two-question gating helper with tests, the extended approve page + view + route, the widened allowlist, and the nudge-tracking write with tests.
</objective>

    - No party row, or token_expires_at < now → 'token_invalid' (the only genuine "expired link" case).

    - sheet.status 'executed' → 'done' (fully executed; offer the signed document).
  </behavior>
