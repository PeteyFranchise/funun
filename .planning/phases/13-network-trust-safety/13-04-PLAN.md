# Plan 13-04: Reporting & Admin Review

<objective>
Let members report profiles, messages, and Green Room content for admin review, then give admins a private queue to review, action, or dismiss reports.
</objective>

<tasks>

<task type="auto">
  <name>Task 1: Add report API</name>
  <acceptance_criteria>
    - Requires authenticated reporter.
    - Supports profile, message, Green Room post, comment, repost, and placement targets.
    - Validates target visibility before report creation.
    - Reported users cannot read report details.
  </acceptance_criteria>
</task>

<task type="auto">
  <name>Task 2: Add admin report queue</name>
  <acceptance_criteria>
    - Admins can filter by status, reason, target type, and date.
    - Green Room content reports can route to hide/remove/pause actions where supported.
    - Admins can mark reviewing, actioned, or dismissed.
    - Internal notes are admin-only.
  </acceptance_criteria>
</task>

</tasks>
