# Plan 12-04: Composer Write API

<objective>
Build `/api/green-room/posts` so users can create structured Green Room posts with safe visibility, linked objects, and draft behavior.
</objective>

<tasks>

<task type="auto">
  <name>Task 1: Add post write service</name>
  <acceptance_criteria>
    - Validates post type, body length, visibility, custom audience, linked object, and resharing preference.
    - Draft posts do not require all publish validations.
    - Published posts require a valid visible linked object when one is supplied.
  </acceptance_criteria>
</task>

<task type="auto">
  <name>Task 2: Add API route</name>
  <acceptance_criteria>
    - Requires authenticated session.
    - Rejects unknown fields and malformed payloads.
    - Returns the created typed card or minimal post payload.
  </acceptance_criteria>
</task>

<task type="auto">
  <name>Task 3: Add adversarial tests</name>
  <acceptance_criteria>
    - Rejects private linked objects, invalid custom audiences, empty published posts, and visibility spoofing.
    - Confirms drafts remain owner-only.
  </acceptance_criteria>
</task>

</tasks>
