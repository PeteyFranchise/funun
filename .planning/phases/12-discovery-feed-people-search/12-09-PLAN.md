# Plan 12-09: People Search & Discover Filters

<objective>
Build the Green Room people discovery layer so members can search and filter public profiles by role, openness, location, genre, and keyword.
</objective>

<product_notes>
People Search is the safe discovery spine of The Green Room. It should expose only public, profile-safe member context and should never become a shortcut around block, visibility, or profile privacy rules. See `12-09-EXECUTION-NOTES.md` for the detailed API shape, filter set, ranking labels, privacy rules, and test plan.
</product_notes>

<tasks>

<task type="auto">
  <name>Task 1: Add discover query service and API</name>
  <acceptance_criteria>
    - Requires authenticated viewer.
    - Excludes blocked and non-public profiles.
    - Supports keyword, role, open-to, location, and genre filters.
    - Uses explicit public-safe profile columns only.
    - Returns reason labels that explain why a person appears.
    - Supports cursor/limit pagination without unstable duplicates.
  </acceptance_criteria>
</task>

<task type="auto">
  <name>Task 2: Add People Search UI</name>
  <acceptance_criteria>
    - Filters are easy to adjust on desktop and mobile.
    - Results show role/capability context without exposing private fields.
    - Empty states suggest broadening filters or exploring featured profiles.
    - Follow/connect/message actions only appear when the viewer can take that action.
    - The Discover tab and right-rail module reuse the same query service.
  </acceptance_criteria>
</task>

</tasks>
