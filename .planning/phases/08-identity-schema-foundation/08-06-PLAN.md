| Boundary | Description |
|----------|-------------|
| admin browser → /admin/members page | Layout redirect is not the authority; each page/route re-verifies is_admin |
| non-admin / anon → POST /api/admin/members | Must be rejected before any account creation |

| invited email inbox → magic link | The link authenticates the invited member on first sign-in |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-08-17 | Elevation of Privilege | Non-admin creating industry accounts via POST /api/admin/members | high | mitigate | verifyAdmin() runs first on both GET and POST; the page independently re-checks is_admin (not layout-only) — matching the established /admin/curators pattern |
| T-08-18 | Tampering | Mass-assignment of unexpected fields / unknown role slugs via the invite POST body | medium | mitigate | strict allowlist validation — only email/display_name/role_slugs accepted; every slug must be in ALL_INDUSTRY_ROLE_SLUGS; createIndustryMember() controls exactly which columns the trigger populates |
