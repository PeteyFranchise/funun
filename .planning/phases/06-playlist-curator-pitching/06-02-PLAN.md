<objective>
Build the admin side of the curator directory: the `/admin/curators` CRUD page (mirroring Phase 5's ChecklistAdmin, minus drag-and-drop), the verifyAdmin-gated CRUD API, the graceful-degradation Spotify/YouTube reach fetchers, the weekly Vercel cron that refreshes reach signals, the shared curator validation schema, and the genre-drift baseline-diff utility (consumed here on admin edit and later by the composer and curator portal).

Purpose: Curators are added one at a time by an admin (D-01) — this is how the directory gets populated. The admin view is also where flag-inactive, drift review/baseline-reset, and claimed/bounced status are surfaced (PITCH-07). Reach signals (PITCH-04's card data) are fetched here and refreshed weekly.
Output: A working admin directory manager, reach infrastructure, and the drift utility other plans import.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin browser → admin curator API | Directory mutations must re-verify admin authority per route |
| Vercel Cron / public internet → cron route | The cron path is publicly reachable; only the CRON_SECRET distinguishes Vercel from an attacker |
| external APIs (Spotify/YouTube) → curators table | Untrusted third-party responses written into reach_signal |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-02 | Denial of Service | `/api/cron/curator-reach` | high | mitigate | `Bearer ${CRON_SECRET}` header check rejected with 401 BEFORE any external fetch; no anonymous trigger of the Spotify/YouTube quota-burning loop |
| T-06-04 | Elevation of Privilege | `/api/admin/curators*` | high | mitigate | `verifyAdmin()` re-checked independently on every handler (not layout-only); writes via service-role client only after the gate passes |
| T-06-06 | Tampering (mass assignment) | admin PATCH curator | medium | mitigate | Update object built strictly from `ADMIN_EDITABLE_FIELDS`; no direct spread of request body |
| T-06-07 | Information Disclosure | external API error surfacing | low | accept | Reach fetchers swallow errors and return null; no third-party error body reaches the client — accepted (best-effort signal) |
</threat_model>
